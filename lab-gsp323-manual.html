TASK 2

Cluster Dataproc e Job Spark

Execução Híbrida
📋 Objetivo Instanciar um cluster Hadoop/Spark via Dataproc, ingerir dados textuais no HDFS do nó principal e disparar um job Spark em lote para calcular a relevância de páginas com o algoritmo SparkPageRank.
⚡ Por que usar o terminal aqui?
  • Erro de subrede: a interface força a subrede default, que o Qwiklabs altera ou remove, travando o provisionamento. A CLI herda a VPC correta automaticamente.
  • Incompatibilidade de imagem: o SparkPageRank usa dependências Scala antigas que quebram no Dataproc 2.2+. A CLI permite travar a versão homologada 2.0-debian10.
Passo 1 — Criar o cluster (Cloud Shell)
bash · Cloud Shell 
gcloud dataproc clusters create spark-cluster \
  --region us-east1 \
  --zone us-east1-b \
  --master-machine-type n2d-standard-2 \
  --master-boot-disk-size 100gb \
  --master-boot-disk-type pd-standard \
  --worker-machine-type n2d-standard-2 \
  --worker-boot-disk-size 100gb \
  --worker-boot-disk-type pd-standard \
  --num-workers 2 \
  --image-version 2.0-debian10 \
  --scopes https://www.googleapis.com/auth/cloud-platform
Passo 2 — Ingerir dados no HDFS (SSH do nó master)

Acesse Dataproc › Clusters › VM Instances e abra o SSH do nó terminado em -m.

bash · SSH master 
hdfs dfs -cp gs://spls/gsp323/data.txt /data.txt
Passo 3 — Submeter o Job Spark (Console Visual)

Acesse Dataproc › Jobs › Submit job e preencha:

  • Regionus-east1
  • Clusterspark-cluster
  • Job typeSpark
  • Main class or jarorg.apache.spark.examples.SparkPageRank
  • Jar filesfile:///usr/lib/spark/examples/jars/spark-examples.jar
  • Arguments/data.txt
TASKS 3 & 4

Chamada das APIs de IA

100% Terminal (Bypass)
📋 Objetivo Task 3: consumir a Cloud Speech-to-Text API para transcrever um arquivo FLAC. Task 4: usar a Cloud Natural Language API para análise de entidades em texto. Ambas as respostas JSON são salvas no Cloud Storage e expostas publicamente para o validador do lab.
⚡ Por que usar o terminal aqui?
  • Cabeçalho de cota obrigatório: chamadas fora do terminal geram erro 403. O curl injeta Authorization e X-Goog-User-Project manualmente, garantindo cobrança na conta efêmera do lab.
  • Bloqueio de política pública (Uniform Access): o Qwiklabs ativa Public Access Prevention por padrão, bloqueando gsutil acl e cliques na interface. A solução é atualizar via IAM no terminal.
Script unificado — cole no Cloud Shell e pressione Enter
bash · Cloud Shell 
# === PASSO A: DEFINIÇÃO DE VARIÁVEIS ===
export PROJECT_ID=$(gcloud config get-value project)
export BUCKET_NAME=qwiklabs-gcp-02-d21a8ff873bb-marking

# === PASSO B: TASK 3 — Speech-to-Text API ===
curl -s -X POST \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "X-Goog-User-Project: $PROJECT_ID" \
  -H "Content-Type: application/json; charset=utf-8" \
  --data-binary '{
    "config": {"encoding":"FLAC","languageCode":"en-US"},
    "audio": {"uri":"gs://spls/gsp323/task3.flac"}
  }' \
  "https://speech.googleapis.com/v1/speech:recognize" \
  > task3-gcs-858.result

gsutil -h "Content-Type: application/json" \
  cp task3-gcs-858.result gs://$BUCKET_NAME/task3-gcs-858.result

# === PASSO C: TASK 4 — Natural Language API ===
curl -s -X POST \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "X-Goog-User-Project: $PROJECT_ID" \
  -H "Content-Type: application/json; charset=utf-8" \
  --data-binary '{
    "document": {
      "type": "PLAIN_TEXT",
      "content": "Old Norse texts portray Odin as one-eyed and long-bearded, frequently wielding a spear named Gungnir and wearing a cloak and a broad hat."
    },
    "encodingType": "UTF8"
  }' \
  "https://language.googleapis.com/v1/documents:analyzeEntities" \
  > task4-cnl-901.result

gsutil -h "Content-Type: application/json" \
  cp task4-cnl-901.result gs://$BUCKET_NAME/task4-cnl-901.result

# === PASSO D: BYPASS DE SEGURANÇA — liberar validador ===
gcloud storage buckets update gs://$BUCKET_NAME --no-public-access-prevention
gsutil iam ch allUsers:objectViewer gs://$BUCKET_NAME
⚠ O que alterar em um NOVO laboratório?
  1. Nome do Bucket (linha 3): substitua qwiklabs-gcp-02-d21a8ff873bb-marking pelo ID do bucket da nova sessão (barra lateral esquerda do Qwiklabs).
  2. Sufixo da Task 3 (linhas com 858): veja no enunciado da Task 3 o nome exato do arquivo de saída e ajuste o número.
  3. Sufixo da Task 4 (linhas com 901): veja no enunciado da Task 4 o nome exato do arquivo de saída e ajuste o número.

A variável PROJECT_ID é detectada automaticamente — não precisa ser alterada.
