GSP313  ·  Compute Engine

Implement Load Balancing
on Compute Engine

Método: 100% Google Cloud Shell (Terminal)

⚠ Antes de começar Substitua os valores de REGION e ZONE na Etapa de Configuração abaixo pelos dados exatos que o Qwiklabs forneceu para a sua sessão ativa (barra lateral esquerda).
⚙️

Requisito: Configurar Variáveis de Ambiente

Abra o Google Cloud Shell e execute os comandos abaixo primeiro. Todas as tarefas seguintes dependem dessas variáveis.

bash · Cloud Shell 
export REGION=# ex: us-east1
export ZONE=  # ex: us-east1-b
TASK 1

Criar Instâncias e Regra de Firewall

📋 Objetivo Criar três VMs (web1, web2, web3) com o Apache pré-instalado via script de inicialização, e liberar a porta 80 no firewall com uma tag de rede.
Recursos que serão criados
vm: web1 vm: web2 vm: web3 firewall: www-firewall-network-lb
Comando unificado — cole no Cloud Shell
bash · Cloud Shell 
gcloud compute instances create web1 \
  --zone=$ZONE \
  --machine-type=e2-small \
  --tags=network-lb-tag \
  --metadata=startup-script='#!/bin/bash
  apt-get update
  apt-get install apache2 -y
  service apache2 restart
  echo "
Web Server: web1
" | tee /var/www/html/index.html'

gcloud compute instances create web2 \
  --zone=$ZONE \
  --machine-type=e2-small \
  --tags=network-lb-tag \
  --metadata=startup-script='#!/bin/bash
  apt-get update
  apt-get install apache2 -y
  service apache2 restart
  echo "
Web Server: web2
" | tee /var/www/html/index.html'

gcloud compute instances create web3 \
  --zone=$ZONE \
  --machine-type=e2-small \
  --tags=network-lb-tag \
  --metadata=startup-script='#!/bin/bash
  apt-get update
  apt-get install apache2 -y
  service apache2 restart
  echo "
Web Server: web3
" | tee /var/www/html/index.html'

gcloud compute firewall-rules create www-firewall-network-lb \
  --allow=tcp:80 \
  --target-tags=network-lb-tag
TASK 2

Configurar o Network Load Balancer

📋 Objetivo Criar um balanceador de carga regional (Layer 4 — TCP/UDP), reservar um IP externo estático, configurar um health check HTTP e anexar as três instâncias criadas na Task 1 ao pool de destino.
Recursos que serão criados
ip: network-lb-ip-1 health-check: basic-check target-pool: www-pool forwarding-rule: www-rule
Comando unificado — cole no Cloud Shell
bash · Cloud Shell 
gcloud compute addresses create network-lb-ip-1 \
  --region=$REGION

gcloud compute http-health-checks create basic-check

gcloud compute target-pools create www-pool \
  --region=$REGION \
  --http-health-check=basic-check

gcloud compute target-pools add-instances www-pool \
  --instances=web1,web2,web3 \
  --instances-zone=$ZONE \
  --region=$REGION

gcloud compute forwarding-rules create www-rule \
  --region=$REGION \
  --ports=80 \
  --address=network-lb-ip-1 \
  --target-pool=www-pool
TASK 3

Criar um HTTP Load Balancer (Layer 7)

📋 Objetivo Configurar um balanceador de carga HTTP global (Layer 7) com grupo de instâncias gerenciado, health check, backend service e regras de roteamento de URL. Execute cada parte separadamente e aguarde a conclusão antes de avançar.
PARTE A Infraestrutura Base — Template, Grupo e Firewall

Cria o template de instância, o grupo gerenciado com 2 VMs e libera o intervalo de IPs do health checker do Google.

instance-template: lb-backend-template managed-group: lb-backend-group firewall: fw-allow-health-check
bash · Cloud Shell 
gcloud compute instance-templates create lb-backend-template \
  --region=$REGION \
  --network=default \
  --subnet=default \
  --tags=allow-health-check \
  --machine-type=e2-medium \
  --image-family=debian-11 \
  --image-project=debian-cloud \
  --metadata=startup-script='#!/bin/bash
  apt-get update
  apt-get install apache2 -y
  a2ensite default-ssl
  a2enmod ssl
  vm_hostname="$(curl -H "Metadata-Flavor:Google" \
    http://169.254.169.254/computeMetadata/v1/instance/name)"
  echo "Page served from: $vm_hostname" | tee /var/www/html/index.html
  systemctl restart apache2'

gcloud compute instance-groups managed create lb-backend-group \
  --template=lb-backend-template \
  --size=2 \
  --zone=$ZONE

gcloud compute firewall-rules create fw-allow-health-check \
  --network=default \
  --action=allow \
  --direction=ingress \
  --source-ranges=130.211.0.0/22,35.191.0.0/16 \
  --target-tags=allow-health-check \
  --rules=tcp:80
PARTE B Saúde e Backend Service

Cria o health check HTTP, define a porta nomeada no grupo e registra o backend service global.

health-check: http-basic-check backend-service: web-backend-service
bash · Cloud Shell 
gcloud compute health-checks create http http-basic-check \
  --port=80

gcloud compute instance-groups managed set-named-ports lb-backend-group \
  --named-ports http:80 \
  --zone=$ZONE

gcloud compute backend-services create web-backend-service \
  --protocol=HTTP \
  --port-name=http \
  --health-checks=http-basic-check \
  --global

gcloud compute backend-services add-backend web-backend-service \
  --instance-group=lb-backend-group \
  --instance-group-zone=$ZONE \
  --global
PARTE C Roteamento e IP Global — Finalização

Cria o URL map, o proxy HTTP, reserva o IP global e cria a forwarding rule que expõe o load balancer na internet.

url-map: web-map-http proxy: http-lb-proxy ip: lb-ipv4-1 forwarding-rule: http-content-rule
bash · Cloud Shell 
gcloud compute url-maps create web-map-http \
  --default-service web-backend-service

gcloud compute target-http-proxies create http-lb-proxy \
  --url-map web-map-http

gcloud compute addresses create lb-ipv4-1 \
  --ip-version=IPV4 \
  --global

gcloud compute forwarding-rules create http-content-rule \
  --address=lb-ipv4-1 \
  --global \
  --target-http-proxy=http-lb-proxy \
  --ports=80
