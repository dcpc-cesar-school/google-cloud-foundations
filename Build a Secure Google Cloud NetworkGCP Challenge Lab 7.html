Build a Secure Google Cloud Network

GCP Challenge Lab - Guia de Execução Completo

TASK 1: Limpeza da VPC

Fase Inicial

Aceda a VPC Network e selecione a rede acme-vpc.

No menu à esquerda, clique em Firewall.

Atenção:

Localize a regra open-access e clique em DELETE.

TASK 2: Preparar o Bastion

Compute Engine

Navegue até Compute Engine > VM Instances.

Na linha da VM bastion, clique nos três pontinhos (⋮) à direita.

Clique em START e confirme no botão azul para iniciar a instância.

TASK 3: Firewall IAP (Bastion)

Segurança
[Referência: Instruções de Firewall IAP - Range 35.235.240.0/20]
Conforme a página 2 do PDF: Use o range indicado para TCP forwarding.

1. Criar Regra de Firewall:

  • Nome: firew1
  • Network: acme-vpc
  • Target Tags: permit-ssh-iap-ingress-q1-326
  • Source IPv4: 35.235.240.0/20
  • Protocols: TCP: 22

2. Editar Instância Bastion:

Vá em VM Instances > bastion > Edit.

No campo Network Tags, cole exatamente a tag abaixo:

permit-ssh-iap-ingress-q1-326

⚠️ Não esqueça de clicar em SAVE ao final da página.

TASK 4: HTTP (Juice-Shop)

Web Traffic

1. Configuração do Firewall:

Nome: firew2
Rede: acme-vpc
Tag: permit-http-ingress-q1-326
IP Source: 0.0.0.0/0
Porta: TCP: 80

2. Configuração na VM:

Clique na VM Juice-Shop > Edit.

Adicione a Tag de Rede:

permit-http-ingress-q1-326

TASK 5: SSH Interno (MGMT)

Rede Privada
Subnet Name Primary IPv4 Range
acme-mgmt-subnet 192.168.10.0/24
Referência da Página 4 do PDF: Copie o IP da Subnet acme-mgmt-subnet.

Nova Regra de Firewall (fw3):

  • Target Tags: permit-ssh-internal-ingress-q1-326
  • Source IP: 192.168.10.0/24 (IP da MGMT Subnet)
  • Portas: TCP: 22

Aplicar no Juice-Shop:

Edite a instância juice-shop e adicione a tag: permit-ssh-internal-ingress-q1-326

TASK 6: Validação Final

Terminal

Abra o SSH da VM bastion e execute os seguintes passos no terminal:

gcloud compute ssh juice-shop --internal-ip
1. Digite Y
2. Pressione ENTER (2x)
3. Digite Y

Se o prompt mudar de bastion para juice-shop, o laboratório está 100% correto!
